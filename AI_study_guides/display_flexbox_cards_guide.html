Display, Flexbox & Cards

The Display Property

Every HTML element creates a "box." The display property answers one question: How does this box behave in relation to other boxes?

Understanding this is the foundation for understanding flexbox, Bootstrap's d-* utilities, and why your original card structure was overcomplicated.

The Five Display Values You Need to Know

Value Behavior Width Default For
block Stacks vertically. Takes full width available. Can set width/height <div>, <p>, <h1>, <section>
inline Sits side-by-side. Only as wide as content. Cannot set width/height <span>, <a>, <strong>
inline-block Sits side-by-side. Only as wide as content. CAN set width/height <img>, <button>
flex Children sit side-by-side with alignment controls. Children can grow/shrink Nothing (you apply it)
none Element disappears completely from the page. N/A Nothing (you apply it)

Bootstrap's Display Utilities

Bootstrap's d-* classes are just shortcuts for setting the CSS display property. They're not magic ‚Äî they're one-line CSS rules.

Bootstrap Class CSS Equivalent What It Does
d-none display: none Hides element completely
d-block display: block Makes element a block (stacks vertically)
d-inline display: inline Makes element inline (sits side-by-side)
d-inline-block display: inline-block Inline but can have width/height
d-flex display: flex Makes children flex items (see Flexbox tab)

Responsive Display Utilities

Add a breakpoint to make display change at different screen sizes:

Your Sidebar Code (from index.html)
<nav class="sidebar d-none d-lg-flex">
  ...
</nav>

  d-none      = display: none (hidden by default)

üîë The Key Insight

Block elements stack. Inline elements sit side-by-side.

When you wanted a logo and company name side-by-side, you used Bootstrap row/col because <div>s are display: block and stack vertically by default.

But you could have just changed the display property ‚Äî either inline-block on the children, or flex on the parent. That's what the Flexbox tab covers.

Flexbox: The Modern Way to Layout

When you put display: flex on a parent element, its direct children sit side-by-side (like inline) but can have width/height (like block). Plus you get powerful alignment controls.

This is what Bootstrap's row uses under the hood ‚Äî but you can use it directly without the column math.

CSS ‚Äî One Line Changes Everything
.parent {
  display: flex;   /* children now sit side-by-side */
  gap: 1rem;       /* space between children */
}

Flex Properties on the Parent

Property What It Controls Common Values
justify-content Horizontal alignment flex-start, center, flex-end, space-between
align-items Vertical alignment flex-start, center, flex-end, stretch
gap Space between children 1rem, 20px, etc.
flex-wrap Whether children wrap to new lines nowrap (default), wrap
flex-direction Direction of layout row (default), column

Flex Properties on the Children

Property What It Controls Common Values
flex-grow How much this child grows to fill extra space 0 (don't grow), 1 (grow)
flex-shrink How much this child shrinks if space is tight 0 (don't shrink), 1 (shrink)
flex-basis Starting width before grow/shrink auto, 200px, 50%
flex Shorthand for grow/shrink/basis flex: 1 = "grow to fill space"

CSS ‚Äî Your Card Header
.card-header {
  display: flex;
  align-items: center;    /* vertically center */
  gap: 1rem;
}

.card-logo {
  width: 80px;            /* fixed size */
  flex-shrink: 0;         /* don't shrink */
}

.card-titles {
  flex: 1;                /* take remaining space */
}

  <div class="col-md-4">

    </div>
  </div>
  <div class="col-md-8">
    <div class="row">

<!-- 5 levels of nesting! -->

Problem: Logo is 33% of row (col-4), not a specific size. Vertical centering is awkward. 5 levels deep.

‚úì Flexbox Version
<div class="card-header">
  <img class="card-logo">
  <div class="card-titles">
    <h3>Name</h3>
    <p>Title</p>
  </div>
</div>



<!-- 2 levels! -->

Better: Logo is exactly 80px. Text fills the rest. Vertically centered with one property. 2 levels deep.

üîë When to Use Flex vs Bootstrap Grid

  • Flexbox: When you have 2-3 items that need to be side-by-side with specific sizing (logo + text, icon + label)
  • Bootstrap Grid: When you have many items that need to reflow into a grid at different breakpoints (card grids, image galleries, skill badges)

The rule: Flexbox for component-level layout. Bootstrap grid for content-level grids.

üîë Card Structure Rules

  • Flexbox for header layout ‚Äî logo + text side-by-side. One display: flex replaces multiple rows and columns.
  • Max 3 levels of nesting ‚Äî Card ‚Üí Header/Body ‚Üí Content. Deeper = rethink the structure.
  • Bootstrap grid only for true grids ‚Äî many repeating items that need to reflow (badges, galleries, feature lists).
  • Name your wrappers ‚Äî if you can't describe what .card_heading_container does that .card-header doesn't, you don't need it.
