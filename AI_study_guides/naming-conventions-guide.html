The Nesting Question

When you have elements nested 3 or 4 levels deep, how do you name them? Do you keep adding prefixes? Let's look at the options.

Option 1: Flat Naming (Recommended)

Name elements by what they ARE, not by their full ancestry. If it's clear from context, you don't need to repeat the parent's name.

.card â†’ the component
.card-header â†’ direct child of card
.company-logo â†’ describes itself â€” no prefix needed
.company-name â†’ describes itself â€” no prefix needed
.job-title â†’ describes itself â€” no prefix needed
.card-body â†’ direct child of card
.card-footer â†’ direct child of card

Notice: .company-logo doesn't need to be .card-header-company-logo. The name is already descriptive enough. You only add prefixes when there's ambiguity.

Option 2: BEM Naming (For Larger Projects)

BEM (Block Element Modifier) uses double underscores to show parent-child relationships. It's more verbose but explicit about structure.

.card â†’ the Block
.card__header â†’ Element of card (double underscore)
.card__body â†’ Element of card
.card__title â†’ Element of card
.card--featured â†’ Modifier (double hyphen) â€” a variant

BEM never goes more than one level: .card__header is valid, but .card__header__logo is NOT. If you need deeper nesting, the inner element becomes its own block or just uses a descriptive name.

ðŸ”‘ Which Should You Use?

  • Flat naming is simpler and works great for small-to-medium projects. Use it unless you have a reason not to.
  • BEM is better for large projects with multiple developers, or when you're building a component library that will be reused across many pages.
  • The most important thing: pick ONE system and use it consistently. Mixed conventions are worse than either system alone.

When Names Might Conflict

What if you have a .title in your card AND a .title in your hero section? Now you have a conflict. Solutions:

Option A: Add Prefix
.card-title { ... }
.hero-title { ... }

Different class names, completely separate styles.

Option B: Scope with Parent
.card .title { ... }
.hero .title { ... }

Same class name, but scoped by parent selector in CSS.

Option A (prefixed names) is usually cleaner. Option B works but increases CSS specificity, which can cause issues later.
