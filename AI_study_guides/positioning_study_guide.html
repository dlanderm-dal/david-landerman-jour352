CSS Positioning: What Actually Happens

The One Thing You Must Understand First

Before we talk about absolute or fixed, you need to understand what they're removing elements from. That thing is called normal flow.

Normal flow is just: elements stack vertically, one after another, each one pushing the next one down. Block elements take full width. That's it. That's the default.

Normal Flow: Elements Stack
header
I'm first. I take up space. Everything after me starts below me.
hero
I'm second. I know the header is above me because it's in the flow.
main content
I'm third. I start right where the hero ends. No overlap. Simple.

The Five Position Values

CSS has five position values: static, relative, absolute, fixed, and sticky. The key question for each: Does it stay in the normal flow, or leave it?

static default

position: static;

This is the default. You never need to write it. Elements stack normally. top, left, right, bottom, and z-index have no effect.

In flow? Yes. Always. That's the whole point.

relative nudge

position: relative;

Element stays in flow and takes up its normal space. BUT you can now use top, left, etc. to nudge it visually from where it would normally be. The space it originally occupied is preserved ‚Äî other elements don't move.

Most common use: Not for nudging at all! It's used to create a positioning context for absolute children.

In flow? Yes! It still takes up space. It just looks shifted.
Relative Creates a Positioning Context
parent (position: relative) anchor point
I don't move. But now my absolute children position relative to ME, not the page.
child (position: absolute) out of flow
top: 50px ‚Äî from parent's top, not page top

absolute removed

position: absolute;

Element is removed from flow. It takes up zero space. Other elements act like it doesn't exist. It positions itself relative to its nearest positioned ancestor (an ancestor with position: relative/absolute/fixed). If none exists, it positions relative to the <html>.

In flow? NO. This is why your content went behind the hero.

fixed viewport-locked

position: fixed;

Like absolute, but positions relative to the viewport (the browser window), not any parent. It doesn't scroll with the page. Stays stuck in place.

Use for: Sticky headers, sidebars, floating buttons, modals.

In flow? NO. Also removed. Also needs you to account for its space manually.

sticky hybrid

position: sticky; top: 0;

A hybrid! Stays in flow like relative, UNTIL you scroll past a threshold (like top: 0). Then it "sticks" like fixed ‚Äî but only within its parent container.

Use for: Section headers that stick while scrolling through that section.

In flow? Yes! It takes up normal space. Best of both worlds.

üîë The Rule

Keep the container in flow. Only pull out the children that need to overlap. The parent having an explicit height while staying position: relative (not absolute) means it still occupies space in the document. The absolute children float inside it, but the parent's box is still "real" to the rest of the page.

Position Types at a Glance

A quick reference for when you're in the middle of coding and need to remember which position type does what.

Position In Flow? Positioned Relative To Scrolls? Use For
static Yes N/A (can't use top/left/etc.) Yes Default. Normal elements.
relative Yes Its own original position Yes Creating anchor for absolute children. Small nudges.
absolute No Nearest positioned ancestor (or html) Yes (with parent) Overlays, tooltips, badges on corners, layered elements.
fixed No The viewport No (stays put) Sticky headers, sidebars, floating buttons, modals.
sticky Yes Scrolls until threshold, then viewport Hybrid Section headers that stick during scroll.

üîë The Golden Rules of Positioning

1. Absolute and fixed remove elements from flow. You must account for the missing space yourself.

2. Absolute children need a positioned parent. Otherwise they position relative to the page. Usually position: relative on the parent.

3. Fixed elements ignore scroll. They're locked to the viewport. Use for persistent UI.

4. Don't put fixed elements inside Bootstrap grid columns. They'll ignore the column anyway and cause layout math to break.

5. Prefer keeping containers in flow. Only use absolute on the specific elements that need to overlap, not the whole section.
