What's happening with float: right

The blockquote slides to the right edge of its container. The paragraphs that come after it in the HTML flow around it on the left side.

Notice: the quote is still "in the flow" enough that text respects its space â€” unlike position: absolute where text would go right under it.

HTML
<article>

  <p>Generic text...</p>

  <blockquote class="pull-quote">
    <p>"Your quote here."</p>

CSS
/* The container */
article {
  font-family: Georgia, serif;
  font-size: 1rem;
  line-height: 1.7;
  color: #333;
  max-width: 800px;      /* Readable width */
  padding: 2rem;
  background: #f5f5f5;   /* Light background */
}

/* Paragraph spacing */
article p {
  margin-bottom: 1rem;
}

/* The pull quote box */
.pull-quote {
  float: right;        /* THE KEY LINE */
  width: 220px;        /* Must set a width */
  margin: 0.5rem;
  margin-left: 1.5rem; /* Space from text */
  padding: 1.5rem;
  background: #ff6b35;
  border-radius: 8px;
  color: white;
}

/* The quote text */
.pull-quote p {
  font-size: 1.1rem;
  font-style: italic;
  margin-bottom: 0.75rem;
}

/* The attribution */
.pull-quote footer {
  font-size: 0.8rem;
  text-align: right;
}

ðŸ”‘ Key Things to Notice

  • You must set a width on the floated element. Without it, the blockquote tries to be full-width and there's no room for text to wrap.
  • margin-left (for float: right) creates breathing room between the quote and the wrapping text. Use margin-right if you're floating left.
  • The HTML order matters. The blockquote should come before the paragraphs you want to wrap around it.
  • No extra divs needed. The paragraphs don't need wrappers â€” they just flow around naturally.
