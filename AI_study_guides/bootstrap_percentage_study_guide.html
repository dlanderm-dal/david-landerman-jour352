CSS Percentages & Bootstrap Structure

The Core Question

How do you get % to work in sizing elements? Do you need to wrap the entire page in a div with width: 100vw? Do you use a Bootstrap container class? What is the acceptable Bootstrap template structure?

1. How CSS Percentages Actually Work

Percentages in CSS are relative to the parent element (technically the "containing block"), not the viewport. This is the fundamental concept that explains why percentages seem "broken" when you don't understand the parent chain.

If you write... It's relative to... Watch out for...
width: 50% Parent's content width Nested containers shrink the reference
height: 50% Parent's content height Parent must have explicit height!
padding: 5% Parent's width (even vertical!) Useful for aspect ratio tricks
margin: 5% Parent's width Same as padding
width: 50vw Viewport width (always) Ignores parent entirely
height: 50vh Viewport height (always) Ignores parent entirely

üîë Key Insight: The Parent Chain

Each level in your HTML creates a new reference point. If you nest a .col-6 inside a .col-8, the inner column's 50% is calculated from the outer column's width ‚Äî not the container.

This is why percentages "work" in Bootstrap ‚Äî each container, row, and column has a defined width, so the math flows down correctly.

% vs vw ‚Äî When to Use Each

vw (viewport width) ignores your parent chain entirely. It always references the browser window. This seems convenient but causes problems when mixed with Bootstrap's percentage-based grid.

‚ö†Ô∏è Don't Mix Width Systems

Using 75vw inside a Bootstrap column creates two competing width systems. The column constrains content via percentages, but vw breaks out of that context. This leads to overflow, alignment issues, and "magic number" hacks.

2. Bootstrap Containers ‚Äî The Foundation

From the official Bootstrap documentation: "Containers are the most basic layout element in Bootstrap and are required when using our default grid system."

The Three Container Types

Class Behavior Use Case
.container Fixed max-width that changes at breakpoints (540px ‚Üí 720px ‚Üí 960px ‚Üí 1140px ‚Üí 1320px) Most content ‚Äî readable width with margins on large screens
.container-fluid Always width: 100% at all breakpoints Full-bleed sections: heroes, footers, backgrounds
.container-{breakpoint} 100% wide until breakpoint, then fixed max-width Hybrid layouts (e.g., .container-lg)

Why Does .container Have Max-Width?

This is a design decision, not a CSS quirk. Reading a paragraph that spans 2500 pixels on a large monitor is exhausting ‚Äî your eyes travel too far, and finding the next line becomes difficult.

The .container caps content at ~1320px and centers it, creating comfortable reading widths even on 4K monitors. This is typography best practice, not a limitation.

What Styles Does .container Actually Have?

/* Bootstrap's .container (simplified) */
width: 100%;
max-width: 1320px;  /* varies by breakpoint */
margin-left: auto;
margin-right: auto;
padding-left: 12px;   /* gutter */
padding-right: 12px;  /* gutter */

The margin: auto is what centers the container. The padding provides the gutters. The max-width controls readability.

3. Mixing Containers on One Page

You are not locked into one container for the whole page. Different sections can use different container strategies. This is the standard convention. 

<body>

    <!-- Full-width hero -->
    <header class="container-fluid">
        ...
    </header>
    
    <!-- Readable content width -->
    <main class="container">
        ...
    </main>
    
    <!-- Full-width footer -->
    <footer class="container-fluid">
        ...
    </footer>

</body>

üîë The Pattern

Each major section gets its own container. Full-bleed backgrounds use .container-fluid. Readable content uses .container. Mix freely based on design needs.

4. Semantic HTML vs Bootstrap Classes

It's important to distinguish between HTML elements and Bootstrap classes ‚Äî they serve different purposes.

Thing What It Is Purpose
<footer> HTML5 semantic element Tells browsers/screen readers "this is a footer"
<header> HTML5 semantic element Tells browsers/screen readers "this is a header"
<main> HTML5 semantic element Marks the primary content area
.mt-auto Bootstrap utility class margin-top: auto
.py-3 Bootstrap utility class Padding on y-axis (top + bottom), size 3
.bg-dark Bootstrap utility class Dark background color

Bootstrap has no "footer component." You use the HTML <footer> element and style it with whatever Bootstrap classes you need.

5. The Bootstrap Grid Hierarchy

Bootstrap's grid has a strict hierarchy. Skipping levels breaks the layout.

Required Structure
.container (or .container-fluid) ‚îî‚îÄ‚îÄ .row ‚îî‚îÄ‚îÄ .col-* ‚îî‚îÄ‚îÄ your content

Do Columns Need Rows? Yes.

From the official Bootstrap documentation: "In a grid layout, content must be placed within columns and only columns may be immediate children of rows."

Element Must Be Inside... Why
.row .container or .container-fluid Container provides width context and horizontal padding
.col-* .row Row provides flexbox layout + negative margins for gutters

What Happens Without a Row?

If you put .col-6 directly inside a .container:

  1. The percentage width works ‚Äî .col-6 is 50%, container has a defined width, math works.
  2. But gutters break. Columns have padding (gutters). Rows have negative margins to compensate. Without the row, content is indented inward and misaligned.
  3. Columns stack vertically instead of sitting side-by-side (no display: flex from the row).

‚ö†Ô∏è The Row's Hidden Jobs

.row does more than you might think:

  • display: flex ‚Äî makes columns sit horizontally
  • flex-wrap: wrap ‚Äî allows columns to wrap to new lines
  • Negative margins ‚Äî pull outward to offset column gutter padding

6. Nesting Grids

You can put a .row inside a .col-* to create nested grids. The inner row treats its parent column as the new container. 

<div class="container">
    <div class="row">
        <div class="col-8">
            <!-- Nested grid -->
            <div class="row">
                <div class="col-6">A</div>
                <div class="col-6">B</div>
            </div>
        </div>
        <div class="col-4">C</div>
    </div>
</div>

In this example, .col-6 inside .col-8 means 50% of 8/12ths of the container ‚Äî creating smaller nested cells.

üîë Nesting Rule

You can nest grids (row inside column), but you cannot skip levels. The hierarchy is always: container ‚Üí row ‚Üí column ‚Üí (optional: row ‚Üí column ‚Üí ...).

    <!-- Fixed elements go OUTSIDE any container -->
    <nav class="navbar fixed-top">...</nav>

        </div>

    <main class="container">
        <div class="row">
            <div class="col-md-8">Main content</div>

        </div>
    </main>

</html>

8. Quick Reference Summary

üîë Answering the Original Question

Do you need to wrap the page in a 100vw div? No.

Do you use a Bootstrap container? Yes ‚Äî containers establish the width context for percentages.

What's the acceptable structure? Container ‚Üí Row ‚Üí Column. Don't skip levels.

Why do percentages "work" in Bootstrap? Because each level (container, row, column) has a defined width, so the percentage chain flows correctly.

The Mental Model

  1. Containers establish width context (and center content)
  2. Rows enable flexbox layout and handle gutter math
  3. Columns divide space using percentages of the row
  4. Percentages always reference the parent ‚Äî not the viewport
  5. vw/vh ignore parents and reference the viewport ‚Äî use sparingly
