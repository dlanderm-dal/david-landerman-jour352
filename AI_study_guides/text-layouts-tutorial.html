The Problem

You want a narrow, readable text column centered on the page. But you also want some elements (images, graphics) to break out wider. How do you structure this?

Approach A: Margins on Each Text Element

Set margin-left and margin-right on every text element (p, h2, ul, etc.). Graphics get no margins and go full-width.

Background Images vs. Content Images

Content images use <img> tags â€” they're part of the document flow. Text can wrap around them (with float) or they push text above/below.

Background images use CSS background-image â€” they're purely decorative. Text completely ignores them and renders on top.

background-size: How Big

Controls the dimensions of the background image. The question is: how big relative to what?

When you set background-size: 80px auto; you're saying:

  • 80px = make the image 80 pixels wide (first value is always width)
  • auto = let the height scale proportionally to maintain aspect ratio

The image doesn't care how big its container is â€” 80px means 80px.

But cover and contain ARE relative to the container:

  • cover = scale the image to completely cover the container (may crop edges)
  • contain = scale the image to fit entirely inside the container (may leave gaps)
auto (default)

contain
background-size: contain;

Fits entirely inside, may leave gaps

80px auto
background-size: 80px auto;

Fixed width, proportional height

Understanding cover vs. contain
cover

Image expands until container is completely filled. Parts of the image may be cropped (hidden outside the container).

contain

Image shrinks/expands until it fits entirely inside. Container may have empty space (background-color shows through).

Making Room for Text (padding vs. position)

Here's where people get confused: background-position moves the image. padding moves the text. They're two separate problems.

Problem 1: Where does the image go?
Solved by background-position

Put the image at the right edge, offset 20px from top, etc.

Problem 2: How do I stop text from overlapping it?
Solved by padding

Push the text content away from the edge where the image sits.

The background image sits behind everything â€” including behind the padding area. So if you add padding-right: 100px, you're creating 100px of space on the right where your text won't go, but the background image can still show through.

CSS
article {
  /* Position the image */
  background-image: url('sidebar-pattern.png');
  background-position: right 10px top 20px;  /* offset from edges */
  background-repeat: repeat-y;
  background-size: 80px auto;  /* 80px wide, height scales */
  
  /* Make room for text */
  padding-right: 100px;  /* wider than the 80px image + offset */
}

ðŸ”‘ Background vs. Float â€” When to Use Which

  • Use background-image for purely decorative patterns that text should ignore. The image sits behind content.
  • Use float (with <img>) when you want text to wrap around the element. The element is content, not decoration.
  • You can use both together: background pattern on the right, floated pull quote on the left.
